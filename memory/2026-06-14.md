# 2026-06-14 日记

## 碰撞盒系统（AABB） — 10:14~10:25

### 改动内容
- **GameState.res**: 新增 `collisionBox = { minX, maxX, minZ, maxZ }` 类型，`playerState` 加该字段
- **ModelConfig.res**: 每个角色类型定义 `animBoxMap`（idle/running 的碰撞盒 @ scale=1.0 基准），`getCollisionBox()` 自动乘以 scale
- **Movement.res**: `isCollision` 改为 AABB 重叠检测（含移动方自身碰撞体积），`applyMove`/`applyIdle` 中动画变化时自动更新碰撞盒
- **PtlSetCollisionBox**: 房间服务新增 API，客户端计算 box 后发送给服务端存储
- **前端**: `IRenderer` 加 `computeWorldBox`/`updateDebugBox`/`removeDebugBox`，`ThreeRenderer` 实现，`MultiplayerLoop` init 时自动计算并发送碰撞盒，`isDebug` 时绘制绿色线框

### 关键决策
- **归一化碰撞盒**: ModelConfig 内所有 box 以 scale=1.0 为基准，`getCollisionBox` 返回时乘以 character scale（原本存了 pre-scaled box 再乘 scale 导致二次缩放 bug，已修复）
- **AABB 重叠检测**: 比较两方碰撞盒是否重叠，非单纯检测点是否在盒内（初版只检查移动目标点 vs 对方盒，未考虑移动方自身体积，已修复）
- **Debug 可视化**: 用 `THREE.EdgesGeometry` + `LineBasicMaterial` 绘制绿色线框矩形，位置与角色同步
- **客户端计算 box**: 使用 `Box3.setFromObject(group)` 获取真实几何体包围盒，通过 `SetCollisionBox` API 发到 room-service

## 碰撞盒重构（删除 SetCollisionBox） — 10:36~11:06

### 问题
1. 巨大娘碰撞盒高度太小 — ModelConfig 默认 box 高度写的是 2（小人的值），巨大娘的实际模型高度 ~15×5=75
2. 小人初始碰撞盒 XZ 位置不对 — `computeWorldBox` 返回世界坐标，但服务端期望相对坐标
3. 巨大娘初始碰撞盒 XZ 太大 — 默认 ±5（base±1 × scale5），实际 MMD 模型没这么宽
4. 碰撞盒有两个来源冲突：客户端 SetCollisionBox（真实几何体 box）vs 服务端 ModelConfig（动画切换更新），导致初始/移动后跳变

### 解决方案
**彻底删除 SetCollisionBox 流程**，碰撞盒完全由服务端 ModelConfig 根据 `characterType + animationName` 控制。

### 改了什么
- **删了：**
  - `PtlSetCollisionBox.ts` + `ApiSetCollisionBox.ts`（room-service）
  - `sendSetCollisionBox`（MultiplayerManager）
  - `computeWorldBox`（IRenderer/ThreeRenderer/MultiplayerRender）
  - `MultiplayerLoop.initForMultiplayer` step 4（compute + send box）
- **修了：**
  - ModelConfig 巨大娘 base box 恢复 `minX/maxX = ±1`
  - `updateCollisionBoxForAnim` 加了契约检查
  - Debug 可视化 `updateDebugBox` 加了 positionY 参数
  - `updateDebugBox` 位置计算 = p.position + box_center_offset（不再只传 x/y）
- **碰撞盒流程：**
  - 初始化 = `getCollisionBox(charType, "idle")`
  - 移动 = `getCollisionBox(charType, "running")`
  - 停下 = `getCollisionBox(charType, "idle")`
  - 初始化后全是服务端 State 广播 + `updateCollisionBoxForAnim`
