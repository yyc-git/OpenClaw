## Step 4: 摄像机 + WASD + 客户端预测（16:26 - 16:50）

### CameraController.ts
- 第三人称跟随，鼠标左键拖拽旋转，滚轮缩放
- 初始 radius 360，无角度限制，far plane 1000
- 自动跟随本地玩家位置（camState.target.lerp）

### WASD 键盘控制
- 替换方向按钮，keydown/keyup 监听
- 按住时 200ms heartbeat 向服务端发指令

### 客户端预测方案演变

**尝试1（基于总经过时间）：** `pos = anchor + dir * 7.5 * (now - anchorTime)`
- 问题：位置漂移，各窗口不一致

**尝试2（per-frame deltaTime 预测 + 3% pullback）：**
- Scene.tsx: `mesh.position += dir * 7.5 * deltaTime`
- LogicManager.update: 3%/帧 拉向服务端 correction target
- Render.ts: 跳过本地玩家
- 问题：30% blend 导致拉扯感，速度变慢

**尝试3（纯插值，无预测）：**
- 两帧 GameState 之间线性插值
- 问题：延迟感

**最终方案（deltaTime 预测 + 预测覆盖）：**
- Scene.tsx: `mesh.position += dir * speed * deltaTime` ← 60FPS 即时移动
- GameState handler: 对所有玩家设服务端位置（覆盖预测）
- 速度匹配：服务端 2.0/tick × 15FPS = 30u/s
- 效果：服务端 15FPS 覆盖 + 60FPS 帧间预测 = 平滑

### 退出修复
- `exit()` 后 await API 响应 → `handleExit` → disconnect + cleanup
- TSRPC 用 `disconnect()` 不是 `close()`
- 进 Hall 时重置 `isOnlyNeedOne/isOnlyNeedTwo`

## 碰撞系统（17:30 - 17:45）

### 服务端
- `_isCollision`: 检查两玩家 XZ 距离 < 1.0
- 碰撞时：位置不变，设 `isCollision = true`
- `_buildSnapshot` 包含 `isCollision` 字段
- 初始位置：`startX = -60, step = 100`（玩家间距 100 单位）

### 客户端碰撞盒
- `createCollisionBox(username, mesh)`: 创建自定义尺寸碰撞盒
  - 巨大娘：4×8×3（宽×高×深）
  - 小人：0.8×1.8×0.5
  - 用 `EdgesGeometry` + `LineSegments` 替代 `BoxHelper`（尺寸可控）
  - 添加到 mesh 子节点 → 自动跟随
- `updateCollisionBox(username, isCollision)`: 绿色/红色切换
- 默认绿色显示
