
## 第三次 Fix：5 个 Bug 修复（12:01-12:10）

### 根因分析 + 修复

**Bug C（核心）：walk 周期扣血不生效**
- 根因：`computeCollisionDamage` 中 `if newTime >= p.walkDuration` — 用的是 player state 的原始值，不是 fallback 后的 `wd`。虽然 `initServerState` 做了 `walkDuration = 0 || 1.0` 的 fallback，但 `p.walkDuration` 在 player state 层面已经是 1.0，所以测试通过。防御式编程不够彻底。
- 修复：改为 `if newTime >= wd`
- 集成测试没抓住的原因：测试中 `p.walkDuration` 经 fallback 后和 `wd` 相等，两条路径结果一样

**Bug 1：collisionBox 高度太矮（只到胸部）**
- 根因：`ModelConfig.res` 中 `giantessBox.maxY = 15`，scale=5 后 75 单位，MMD 模型实际更高
- 修复：`maxY: 15 → 20`（scale=5 后 100 单位 ≈ 头顶）

**Bug 2：血条旋转时抖动**
- 根因：`adjustAllHpBarPositions` 用 `if (ndcY !== vec.y)` 条件判断 — 相机旋转时 NDC Y 在边界附近进出，clamp toggle
- 修复：去掉 if 条件，每帧全量 project→clamp→unproject→set(xyz)

**Bug 3：拉近时血条太大**
- 根因：barWidth = AABB 宽度（giantess 达 20），无上限
- 修复：`Math.min(barWidth, 10)`

**Bug 4：贴脸时血条消失**
- 根因：相机太近时 sprite 在 NDC Z 超出 [-1, 1]
- 修复：NDC Z 检查，超出则跳过不调整
