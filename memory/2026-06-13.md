## MMD 加载 Bug 修复

- `_getValidExpressions()` 修复：loader 前设 `globalThis.validExpressions = null`
- 纹理 404 修复：复制 data/ 目录

## 脚部 IK 问题（2026-06-13）

### ✅ 最终方案
- 修改 `packages/meta3d-jiehuo-abstract/src/three/MMDAnimationHelper.js` 的 `_animatePMXMesh`
- 循环 IK 前：保存 mesh.transform → 归零 → `mesh.updateMatrixWorld(true)` → 跑 IK → 恢复
- IK 在 mesh-local 空间解算，不受 setTransform 影响

### 失败的方案
1. **Group 套一层** — 无效，bone.updateWorldMatrix(true,true) 走父链包含 Group
2. **MMDManager.update 归零** — 第一帧 IK 就错
3. **_animatePMXMesh 归零但不更新 matrixWorld** — 骨骼 matrixWorld 不一致

### 关键发现
- meta3d 将 `mesh.updateMatrixWorld(true)` 替换为只更新 右足/左足的 `_updateBoneMatrixWorldForIK`
- 其他骨骼 matrixWorld 可能是 stale 值

## 移动朝向问题（2026-06-13）

### 最终方案
- `handleCubeNotMove` 不再改变 `rotationY`（保持最后一次移动朝向）
- `applyMove` 继续用 `atan2(moveX, moveZ)` 计算移动朝向
- 空闲时相机旋转不追踪模型（视觉层面待优化）

### 待办
- 空闲时相机旋转应带动模型转向（当前仅靠 handleCubeNotMove 改 rotationY，现已禁用）

## 飞书通知方式（2026-06-13 最终确认）

### ✅ 可用方式（cron + delivery.announce）
- `cron add` 创建一次性 job
- `delivery.mode: "announce"`, `delivery.channel: "feishu"`, `delivery.to: "user:ou_..."`（**必须带 `user:` 前缀**）
- `sessionTarget: "isolated"`, `deleteAfterRun: true`
- 通知内容放在 `payload.message` 中
- ⚠️ 通知内容不宜过长（<30字），否则飞书回复消息可能干扰流程

### ❌ 不可用方式
- `sessions_send` 到 feishu direct session → delivery 卡在 pending，不会投递
- `cron` + `delivery.announce` 不带 `to` 或 `to` 没有 `user:` 前缀 → 投递失败

## 通知系统排坑（2026-06-13 18:30）

### sessions_send 到飞书
- `sessions_send` 到 feishu direct session **实际能投递到飞书 App**
- 但会触发 bot 自动回复一条消息（如「收到兄弟」）
- 过去认为卡 pending = 不投递，实测 pending 状态不影响投递
- ⚠️ bot 回复可能对会话流造成干扰，短期可接受

### cron 调度器卡死
- `cron add` 创建的一次性 job 有时卡在 running 状态不执行
- 表现为：用户没收到通知，但 cron job 列表显示 running
- 重启 Gateway 可解决调度器卡死
- 重启后之前运行的 cron job 会报 LLM request failed（残余 job）

## 重构：MMD 职责从 ThreeRenderer 下沉到 MultiplayerRender（2026-06-13 18:40）

### 背景
- ThreeRenderer 不应包含跨淡等 MMD 逻辑，应只做原子渲染
- logic_layer 不应直接操作渲染，应通过 IRenderer 接口

### 改动
| 文件 | 操作 | 原因 |
|------|------|------|
| `ThreeRenderer.ts` | **改** — `crossFade` 拆成 `fadeOutAction`/`fadeInAction`/`resetAction` | ThreeRenderer 只保留原子操作 |
| `IRenderer.ts` | **改** — 加 `fadeOutAction`/`fadeInAction`/`resetAction` | 接口暴露原子动画操作 |
| `MultiplayerRender.ts` | **改** — `crossFade` 用原子 ops 组合实现 | 跨淡逻辑归渲染层 |
| `Operate.tsx (giantess)` | **改** — `d3` 加 `as unknown as Array<Array<singleOperateData>>` | 修复预存 TS 错误 |
| `Operate.tsx (little_man)` | **改** — 同上 + 修正类型嵌套 + `imageBase64` cast | 修复预存 TS 错误 |

### 架构图
```
MultiplayerRender (IRenderer)
  ├─ crossFade (组合 fadeOutAction + fadeInAction + resetAction)
  ├─ createMMDCharacter (MMDLoader + addSceneNode)
  ├─ updateMMDAnimations (helper.update)
  └─ 委托到 ThreeRenderer (setPosition, render, init...)

ThreeRenderer (原子渲染)
  ├─ fadeOutAction / fadeInAction / resetAction
  ├─ addEntity / removeEntity / setPosition
  ├─ addSceneNode / removeSceneNode
  └─ render / init / dispose

IRenderer (契约接口)
  └─ logic_layer 只依赖此接口
```

### 结果
- **TypeScript 编译 0 错误** ✅（修复了预存的 Operate.tsx 编译错误）
- **全部 6 suites / 45 BDD 测试通过** ✅
