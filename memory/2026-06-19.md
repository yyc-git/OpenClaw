
### 第5轮（最终）：使用自定义 Sprite 类 ✅
**现象:** 蓝色 Mesh 可见，所有 Sprite（血条+红绿黄诊断）不可见
**诊断:** `calls` 差 2 —— Sprite 不产生 draw call，Three.js SpritePlugin 渲染空列表
**根因:** THREE.Sprite 共享 `_geometry`（InterleavedBuffer + InterleavedBufferAttribute）的 `_glAttribute`/`_glBuffer` 残留旧 WebGL context 的 buffer ID。新 renderer 检查 `attribute._glAttribute` 发现已存在（指向旧 context），不复传到新 context → Sprite 无 draw call
**修复:** 使用 `meta3d-jiehuo-abstract/src/three/Sprite.js` 自定义 Sprite：
- `dispose()` 把共享 `_geometry = undefined`
- `clearAllSprites` 中调用 `sprite.dispose()`
- `ThreeRenderer.dispose()` 加 `forceContextLoss()` + program/info 清理
**验证:** 双局血条和诊断 Sprite 都正确渲染 ✅
