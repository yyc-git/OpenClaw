### 看不到另一个玩家的方块
- **原因：** `createEntity` 后没设初始位置，叠在 (0,0,0)

- **修复：** 所有 `Math.atan2(dx, dz)` 改为 `Math.atan2(dx, -dz)`
- 涉及文件：

  - `packages/frontend/src/render_layer/MultiplayerRender.ts` — 本地玩家旋转
  - `packages/frontend/src/render_layer/ThirdPersonCamera.ts` — 空闲时面向相机

### 停止移动时方块 snap 回 0°
- **原因：** `CubeNotMove` 把 `rotationY` 强制赋 0
- **修复：** 不重置 rotationY，保持最后移动方向

## Bug 修复（内存密集，17:00~21:00）

### 内存清理
- 发现内存接近满了
- 杀掉了两个卡死的 npm install 进程（pid 6484 和 15892），释放 ~2.9GB

### 1. 旋转反了
- **原因：** Three.js 中 `rotation.y=0` 时 local +Z 指向 world +Z，但 `getCameraLookAngle` 用的是相机朝向场景的方向，需要面朝相机
- **修复：** `getCameraLookAngle` 改为从场景指向相机的方向；`ManagerRender.ts` 里保留 `atan2(dx, dz)`（与 Three.js 一致）

### 2. 侧面角度时移动方向偏斜
- **原因：** 客户端传 4 个布尔值（`isMoveLeft/Right/Forward/Backward`），当相机在某个角度时 W 的世界空间移动同时有 X 和 Z 分量→两个布尔值都为 true→服务端误当 45° 对角线
- **修复：** 改为传 `{ moveX, moveZ }` 浮点数，服务端直接用 `dx = moveX * speed`
- 改的文件：
  - `packages/logic/src/logic/Movement.ts` — 改用 `moveX/moveZ`
  - `packages/frontend/src/business_layer/Loop.ts` — 客户端发 `moveX/moveZ`
  - `packages/logic/test/step-definitions/movement.steps.ts` — 更新测试数据
  - `packages/logic/test/features/movement.feature` — 更新对角线期望值

### 3. 服务端 JS 编译产物未更新（坑！）
- **原因：** `packages/logic/src/Movement.js` 是旧编译（18:40），服务端通过 `require("../../../logic/src")` 加载时用了旧代码。新版实际编译到了 `packages/logic/dist/`，但 `packages/logic/src/*.js` 还是旧的
- **修复：** 用全局 `tsc` 手动编译 logic 包，把生成的 `.js` 文件拷贝到 `packages/logic/src/` 对应位置
- **注意：** 以后改完 `packages/logic/src/*.ts` 必须确认 `.js` 也重新编译到位

### 4. 启动脚本
- yarn workspace 和 npm workspace 因 `demos/` 与 `packages/` 同名包冲突（room-service、match-service）
- 最终修复：兄弟删除了 `node_modules` 和 `yarn.lock` 后重新 `yarn bootstrap`
- 前端启动：`cross-env NODE_ENV=development webpack-dev-server --config webpack.config.devserver.js`
- 服务端启动：先 `tsc --project tsconfig.build.json` 编译，再 `node dist/index.js`
