## [2026-06-10 11:25] yarn bootstrap 优化分析

### 问题
每次增删改项目后跑 `yarn bootstrap` 太慢（>1min），因为会遍历整个 monorepo。

### 方案：node_modules junction
如果新增的项目没有/少有外部 npm 依赖（纯源码），可以直接在根 `node_modules/` 建 junction 链接到该项目目录，跳过 yarn/lerna 的依赖解析。

```bash
cd D:\Github\GTS-Play\node_modules
mklink /J 项目名 ..\packages\项目名
```

### 适用条件
- ✅ 新项目无外部 npm 依赖（或依赖极少）
- ✅ 需要编译产出（如 ReScript 的 `rescript build`）
- ❌ 不适用：项目有大量外部 npm 依赖
- ❌ 不适用：需要 yarn workspace 自动处理依赖冲突

### 实施步骤
1. 在根 `node_modules/` 建 junction
2. 如果项目需要编译，直接进去跑编译命令
3. 如果有外部依赖，需要先在项目目录跑 `npm install` 或 `yarn install`
